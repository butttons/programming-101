              # Node starter

              This will guide you through setting up MySQL on your system and a basic directory structure for your app. The code can be found in the tutorials/node-starter folder of this repo.

              # MySQL Setup

              • Easiest way to manage MySQL databases locally is by using phpMyAdmin. It comes with most php based web servers.
              • Web servers you can install easily. -
                • XAMPP - Linux, Mac, Windows
                • WAMP - Windows
                • MAMP - Mac
                • LAMP - Linux, Mac, Windows
              • When you're done installing you can start the service and you localhost should be live. Normally it's on default 80 port, and 8888 port on MAC.
              • When installing, make sure you save the MySQL credentials. In case you've misplaced them, follow this guide to reset your password. Or just simply create a new user by following this guide. Make sure the user you create has all privileges.
              • Open phpMyAdmin by going to http://localhost:8888/phpmyadmin/ or http://localhost:8888/phpmyadmin/ and login using the above credentials.
              • Locate Database on the top bar, and create a new database, which we will be using for our project. Something short and apt.
              • We will be needing the username, password and database name later on. Keep those handy.

              # Nodejs project setup

              Now that our MySQL database is up and running, we can start coding.

              • Create a new directory and initialize a package.json.
              • Installing dependencies. We will be using the following -
                • knexjs - SQL query builder.
                • express - Web framework.
                • body-parser - Express middleware plugin for parsing request body.
                • mysql - mysql package for node.
                • dotenv - Load env variables from .env files.
                npm install knex express body-parser --save
              • Install these dependencies too to make your development cycle better -
                • nodemon - Utility to restart your server on file changes.
                npm install nodemon --save-dev
              • Create server directory. This will house all our server code.

              # Boilerplate code and directory structure.

              • Create a .env file in the root of our project with these contents:
              DATABASE_HOST='localhost'
DATABASE_PORT='3306'
DATABASE_USERNAME='--db username from mysql setup--'
DATABASE_PASSWORD='--db password from mysql setup--'
DATABASE_NAME='--db name from mysql setup--'
NODE_ENV='dev'
              • Start by creating an app.js in server folder. This will be our main express app file.
              const express = require('express');
const api = require('./api');

const app = express(); // Initialize express
app.use('/api', api); // Mount our api on /api

// Simple text greeting on /
app.get('/', (req, res) => {
    res.send('Hello world');
});

module.exports = app;
              • Create an index.js. This will just create a server on port and mount our express app on it.
              const http = require('http');
const app = require('./app'); // Get our express app

const port = 4000;
const server = http.createServer(app); // Create a server
// Start server
server.listen(port, () => {
    console.log(`Server started on http://localhost:${port}/`);
});
              • Create an api folder and index.js file in it.
              const router = require('express').Router(); // Get express router
const authRoute = require('./auth'); // Get our /api/auth handle

router.use('/auth', authRoute); // Mount on /api/auth

module.exports = router;
              • Create an auth folder in the api folder, with index.js in it.
              const router = require('express').Router();

// Handle at GET /api/auth/get_code
router.get('/get_code', (req, res) => {
    res.send('Dummy GET request on /api/auth/get_code');
});

module.exports = router;
              • Now your directory should look something like this:
              ├── package.json
├── package-lock.json
└── server // Main folder for our server
    ├── api // Main folder for our api
    │   ├── auth
    │   │   └── index.js // auth handle for our api. This folder can house various other routes too to use in /api/auth.
    │   └── index.js // Main file for our api. All our various routes can be merged here.
    ├── app.js // Express app
    └── index.js // Server start point
              • Add the following 2 scripts to your package.json in the scripts section:
              "scripts": {
    ...
    "server:start": "node server/index.js",
    "server:dev": "nodemon server/index.js"
}

              # Database setup

              • Start by creating a database folder in the server folder.
              • Create a db.js. This file will export the authenticated knexjs database instance. Which will be used by our functions.
              require('dotenv').config();
const knex = require('knex')({
    client: 'mysql',
    connection: {
        host: process.env.DATABASE_HOST,
        port: +process.env.DATABASE_PORT,
        user: process.env.DATABASE_USERNAME,
        password: process.env.DATABASE_PASSWORD,
        database: process.env.DATABASE_NAME
    },
    debug: process.env.NODE_ENV === 'dev' // Turns on debugging for our development.
});
module.exports = knex;
              • Create a schema.js file. This is responsible for creating and setting up the tables.
                • schema.createTable
                • column.refrences
              const db = require('./db');
(async () => {
    try {
        await db.raw('SET foreign_key_checks = 0;'),

        await db.schema.dropTableIfExists('users');
        await db.schema.dropTableIfExists('games');
        await db.schema.dropTableIfExists('gamelist');

        await db.schema.createTable('users', (table) => {
            table.increments();
            table.string('name');
            table.string('reddit_name');
            table.string('reddit_token');
            table.timestamps();
        });

        await db.schema.createTable('games', (table) => {
            table.increments();
            table.string('name');
            table.string('genre');
            table.string('platform');
            table.string('website');
            table.timestamps();
        });

        await db.schema.createTable('gamelist', (table) => {
            table.increments();
            table.integer('user_id').unsigned().notNullable().references('id').inTable('users').onDelete('cascade');
            table.integer('game_id').unsigned().notNullable().references('id').inTable('games').onDelete('cascade');
            table.string('game_handle');
            table.time('time_start');
            table.time('time_end');
            table.string('day_name');
        });
    } catch (err) {
        console.error('Error creating database schema', err);
    }
})();
              • Update package.json for easy use of the schema generation file.
              "scripts": {
    ...
    "db:schema": "node server/database/schema.js"
}

              # Database usage

              • Create a handles folder in your database folder.
              • Create an index.js file in handles folder.
              const db = require('../db');
const handles = {
    games: {
        addNewGame: async ({ name, genre, platform, website }) => {
            const row = await db.table('games').insert({
                name,
                genre,
                platform,
                website
            });
            return row;
        }
    }
};
module.exports = handles;
              • Create a handles.sandbox.js file in handles folder.
              const handles = require('./index');
(async () => {
    const newGame = {
        name: 'Battlefield V',
        genre: 'shooter',
        platform: 'pc',
        website: 'http://www.battlefield.com/'
    };
    const out = await handles.games.addNewGame(newGame);
    console.log('output', out);
})();
              • Now you can test your new handle for the database easily. Just run node server/database/handles/handles.sandbox.js
              • You can organize your database function by creating more folders, grouping the functions by their features.
              • Your final directory structure should be something like.
              ├── package.json
├── package-lock.json
└── server
    ├── api
    │   ├── auth
    │   │   └── index.js
    │   └── index.js
    ├── app.js
    ├── database
    │   ├── db.js
    │   ├── handles
    │   │   ├── handles.sandbox.js
    │   │   └── index.js
    │   └── schema.js
    └── index.js

              # Using NPM scripts

              • npm run server:start - This starts the server for once.
              • npm run server:dev - This starts the server in watch mode. Any changes made to files will automatically restart the server.
              • npm run db:schema - This will attempt to delete all exisiting tables and create fresh new ones.
